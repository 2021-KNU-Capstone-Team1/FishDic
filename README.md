# 현재 레포지토리를 로컬 컴퓨터로 받는 방법
1) 소스트리를 열고 Clone 클릭
2) 소스 경로에 https://github.com/2021-KNU-Capstone-Team1/FishDic.git 입력
3) 현재 컴퓨터에서 저장 할 폴더 지정 후 클론 클릭
 
# 로컬 컴퓨터에서 깃허브로 올리는 방법
1) 소스트리를 열고 만약 새롭게 추가하거나 수정한 사항이 존재한다면
2) 스테이지에 올라가지 않은 파일들 목록이 뜬다.
3) 모두 스테이지에 올리기를 누른다음
4) 아래 쪽에 커밋 버튼이 있는 곳에 보면 메모 남기는 곳이 있다.
5) 거기에 커밋 메시지를 적당히 남기고 커밋 버튼을 누른다.
6) origin/master에 즉시 푸시라는 체크박스가 있는데 남들도 뜨는지는 모르겠는데 걍 체크하고 커밋하면 됨
7) 중요 : 만약 그럴일이 있을지는 모르겠는데 커밋 눌렀는데 충돌났다고 뜨면서 안된다면
깃허브 사이트에 올라가 있는 파일하고 현재 로컬 컴퓨터의 파일에서 충돌난 부분을 일일이 찾아서 수정해야 함
8) 왜 충돌이 발생하는가? A라는 사람이 깃허브에서 로컬 컴퓨터로 Clone을 했을 때, B라는 사람이 깃허브에 뭔가를 변경해서 올렸다고 치면
A의 로컬 컴퓨터에는 B가 변경한 사항이 반영이 안되어있기 때문에, A가 깃허브에 올리려고 할 때, 서로 충돌이 발생한다.
이를 방지하기 위해서는, 항상 작업 전에 누가 새롭게 올린게 있나 확인해야 함 (패치 버튼)
아니면, 브랜치를 나눠서 하는 방법도 있는데 이거도 충돌 발생하면 해결하기 존나 복잡하니까 걍 단순하게 마스터 브랜치 하나만 가지고 하자
